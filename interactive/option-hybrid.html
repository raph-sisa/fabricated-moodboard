AI-Assisted Hardware Development - 8 Lessons

Lesson 1: Getting Started

Claude Code is your AI pair programmer for hardware projects. It understands Arduino sketches, ESP-IDF configurations, circuit schematics, and can help you debug serial output in real-time. This module teaches you how to leverage Claude Code from first prompt to finished firmware.
bashGetting started
# Install Claude Code CLI
npm install -g @anthropic-ai/claude-code

# Start a session in your project directory
cd ~/my-arduino-project
claude

# Ask Claude to help with your sketch
> "Help me write a sketch that reads a BME280
   sensor over I2C and displays on an SSD1306 OLED"

Lesson 2: Debugging Serial Output

One of Claude Code's strengths is parsing messy serial output. Paste your Serial Monitor logs directly into the conversation and Claude can identify timing issues, buffer overflows, and sensor misreadings that would take hours to track down manually.
c++Debug-friendly serial output
void debugSensor(float temp, float hum) {
  Serial.print("[SENSOR] t=");
  Serial.print(temp, 2);
  Serial.print(" h=");
  Serial.print(hum, 2);
  Serial.print(" ts=");
  Serial.println(millis());
}

Lesson 3: Generating Pin Configurations

Let Claude Code generate your pin definitions and initialization code. Describe your circuit in natural language and get back properly structured header files with documentation and safety checks built in.

  constexpr uint8_t LED_G      = 26;
  constexpr uint8_t LED_B      = 27;
  constexpr uint8_t BTN_PIN    = 13;
}

Lesson 1: Blink

Every hardware journey begins with Blink. It's the "Hello World" of physical computing - a single LED turning on and off. Simple, but profound. This first program proves your toolchain works, your board is alive, and you can make something happen in the real world with nothing but code.
arduinoBlink.ino
// The classic Arduino Blink sketch
// Built-in LED on most boards is pin 13

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  delay(1000);  // on for 1 second
  digitalWrite(LED_BUILTIN, LOW);
  delay(1000);  // off for 1 second
}
Upload this sketch and watch the LED blink. Then try changing the delay values. What happens at delay(100)? At delay(50)? This is your first experiment with timing in embedded systems.

Key Concepts

pinMode() configures a pin as INPUT or OUTPUT. You must call this before reading or writing to a pin.

digitalWrite() sets a pin HIGH (5V/3.3V) or LOW (0V). This is binary - full on or full off.

delay() pauses execution for N milliseconds. Simple but blocking - in later lessons we'll learn non-blocking alternatives with millis().

Projects

Build these. Ship these. Learn by making real things.

Weather Station

Beginner ~4 hours ESP32 + BME280 + OLED
Build a complete weather monitoring station that reads temperature, humidity, and barometric pressure from a BME280 sensor, displays real-time data on a 128x64 OLED, and logs to an SD card. Optional: add WiFi for remote monitoring via a simple web server.

PARTS LIST

  • ESP32 DevKit V1
  • BME280 breakout (I2C)
  • SSD1306 OLED 128x64
  • Micro SD card module
  • Breadboard + jumper wires
  • USB-C cable

MIDI Controller

Intermediate ~8 hours Teensy 4.0 + Pots + Buttons
Design and build a custom USB-MIDI controller. Four rotary potentiometers for continuous control, eight tactile buttons for note triggers, and RGB LED feedback. Uses Teensy's native USB-MIDI - no drivers needed, just plug in and map to your DAW.

PARTS LIST

  • Teensy 4.0
  • 4x 10K linear potentiometers
  • 8x tactile push buttons
  • 4x WS2812B LEDs
  • Perfboard or custom PCB
  • Enclosure (3D printed)

LED Matrix Display

Beginner ~3 hours Arduino Nano + WS2812B
Create a sound-reactive LED display using a strip of WS2812B addressable LEDs and an electret microphone module. The Arduino performs a simple FFT to split audio into frequency bands and maps them to LED colors and brightness. Perfect for desk ambiance or party lighting.

PARTS LIST

  • Arduino Nano
  • WS2812B strip (60 LEDs/m)
  • MAX9814 mic module
  • 5V 3A power supply
  • 470 ohm resistor
  • 1000uF capacitor

Sensor Array Logger

Intermediate ~6 hours Arduino Uno + Shield
Build a multi-sensor data acquisition system on a custom prototyping shield. Combines a PIR motion sensor, photoresistor, and DHT22 temperature/humidity sensor. Data logs to SD card with timestamps from an RTC module. Great for environmental monitoring or security projects.

PARTS LIST

  • Arduino Uno R3
  • Proto shield + headers
  • HC-SR501 PIR sensor
  • Photoresistor + 10K resistor
  • DHT22 sensor
  • DS3231 RTC module
  • SD card module
  • Breadboard for prototyping
